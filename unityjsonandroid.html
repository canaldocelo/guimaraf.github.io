Unity - Carregando o json da pasta StreamingAssets no Android

By Fred Oliveira sobre Unity
22 de setembro de 2020

Um grande problema quando queremos usar um arquivo json para por exemplo traduzir um aplicativo/jogo, no Android não conseguimos acessar a pasta StreamingAssets diretamente. A pasta StreamingAssets é uma pasta que o Unity não criptografa, sendo excelente para colocar arquivos de tradução ou até arquivos de configuração do seu aplicativo/jogo.
Depois de algumas pesquisas, descobri como resolver este problema relacionado ao Android, quero deixar registrado para ajudar quem tiver dificuldades. A pasta StreamingAssets não é criada por padrão nos projetos, então na raiz do seu projeto.
Primeiro vamos entender o funcionamento da leitura do json na game engine Unity.
Existem diversos métodos para fazer a leitura de um json na engine, dos que tive a oportunidade de estudar, eu considero este como o mais simples. Vamos construir um json bem simples, bem direto ao ponto, exatamente o que preciso em uma aplicação que estou desenvolvendo no momento.
Na pasta StreamingAssets, crie o arquivo "pt.json", você pode criar com qualquer nome, mas para este exemplo, lembre que vamos tratar o nome do arquivo como "pt.json".
Veja o json abaixo. 

{
    "gameName": "Nome do jogo",
    "gameMode1": "Modo de jogo 1",
    "gameMode2": "Modo de jogo 2",
}
Criado o json, mo Precisamos montar a classe base que a engine vai acessar diretamente.

public class Translate : MonoBehaviour
{
    public TranslateDB translateDB = default;
    void Start()
    {
        translateDB = new TranslateDB();
        translateDB.LoadJson();
    }
}
Depois vamos montar a classe que contém as variáveis com os mesmos nomes das keys do arquivo json, coloquei o nome de TraslateDB.

public class TranslateDB
{
    public string gameName = default;
    public string gameMode1 = default;
    public string gameMode2 = default;
}
Dentro da mesma classe TranslateDB crie o método que faz a leitura do json

public void LoadJson()
{
    string fileName = "pt.json";
    string filePath = Application.streamingAssetsPath + "/" + fileName;
    string jsonString = "";
                                    
    jsonString = File.ReadAllText(filePath);
    var jsonData = JsonUtility.FromJson(jsonString);
    gameName = jsonData.gameName;
    gameMode1 = jsonData.gameMode1;
    gameMode2 = jsonData.gameMode2;
}
Obs¹ Infelizmente o código acima acabou recebendo uma alteração não intencional, pois o HTML5 reconhecer o <TranslateDB> como uma tag HTML e tenta fechar a tag no final do código, gerando o </translatedb>.

Ignore o final do arquivo e copie o código apenas até o último }

Obs² Na seguinte linha
var jsonData = JsonUtility.FromJson<translatedb>(jsonString);

Onde está o translateDB, na verdade deveria ser TranslateDB, que é o nome da própria classe. Considere como está entre <> o HTML5 já converte para translateDB, substitua, ficando desta forma

var jsonData = JsonUtility.FromJson<TranslateDB>(jsonString);

TranslateDB Continuando, a método de acesso a arquivos da pasta StreamingAssets é a padrão da engine, como já mencionamos, no Android, o json está dentro do apk, não sendo possível ter acesso diretamente, precisando fazer um ajuste no método de leitura para que seja verificado dentro do apk pela existência do arquivo.json especificado.

public void LoadJson()
{
    string fileName = "pt.json";
    string filePath = Application.streamingAssetsPath + "/" + fileName;
    string jsonString = "";

    if (Application.platform == RuntimePlatform.Android)
    {
        WWW reader = new WWW(filePath);
        while (!reader.isDone) { }
        jsonString = reader.text;
    }
    else
    {
        jsonString = File.ReadAllText(filePath);
    }
    var jsonData = JsonUtility.FromJson(jsonString);
    gameName = jsonData.gameName;
    gameMode1 = jsonData.gameMode1;
    gameMode2 = jsonData.gameMode2;
    buttonBack = jsonData.buttonBack;
}
Desta forma, você consegue realizar a tradução também no Android
Obs³ Lembre de substituir por esta linha

var jsonData = JsonUtility.FromJson<TranslateDB>(jsonString);
