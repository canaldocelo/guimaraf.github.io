Generating the Xcode project on the Mac using Cordova CLI

Cordova CLI to IOS

I'm starting from the point where your environment is all configured, Xcode installed, Node, Cordova, already have the developer account of Apple and also the game already exported by Construct 2, if you have not done, check the first part I wrote.

We will address until the creation of the Xcode project, thus eliminating the need for some service like Cocoon or Phonegap Build, which we also discussed in the previous article.

Now I need to send the files exported from Construct 2 to my Mac, I usually use Google Drive via browser, since I can just drag and drop the file in the browser, that the upload is done automatically, then I just need to log my account on the MAC and download the file.
Send the file in the formed .ZIP, because the OSX itself will unpack the file, creating a folder with the same name.
With the project files on the MAC, we need to understand some terminal commands, see in this link the main commands, however we will use only a few.

ls shows the list of all your files and folders
cd allows you to go from one directory to another (ex: cd / user / yourname to go to your directory)
clear clears the screen, leaving only the first line
The cd .. command is used to go to the previous directory or exit the current directory

Preparing the project

This image is from the directory exported by Construct 2, as you can see it has only the project files and the configuration files.

With the terminal open, I used the ls command, showing the same files by the terminal.

Now we can prepare the IOS platform, with the command

cordova platform add ios

If you want to insert a specific version of the IOS Cordova, it is necessary to specify the version. Remember that the current version is 4.5.4, the command would be.

cordova platform add ios@4.5.4

Cordova will analyze the configuration files and install the described plugins.

I put an arrow to show the command typed, I left showing all the others until the end of the installation of the IOS platform and the plugins.
You can now use the clear command to clear the screen.

Okay, now you already have your Xcode project generated. Open Xcode and navigate until you find the required configuration file directory, see the image below.

Wait for Xcode to finish parsing the files.

We completed the entire process from the project generation in Construct 2 to the Xcode project generation, the rest of the process to send to the Apple store I will not address.

If you want to use ads in projero, see the steps below, which is approached the installation of the plugin.

For ADS I am using the Ultimate ADS plugin, which is already inserted in the project.

It is necessary to manually insert the plugin into the platform, use the command below.
Cordova plugin add cocoon-plugin-ads-ios-admob


With the plugin installed, regenerate the Xcode project.
